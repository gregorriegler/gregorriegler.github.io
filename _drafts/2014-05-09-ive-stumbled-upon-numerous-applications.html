The Jpa Dependency dilemma

Web -> Application -> Dao

It works, and it's also not wrong, but it's flawed. Why? Because you can't simply exchange the dao. You had to change your application layer in order to exchange the dao. This smells like bad design. Obviously, it should be easy to exchange a low level detail like how data would be persisted.

We can improve the design by inverting the dependency between the application layer and the dao like this.

Web -> Application <- Dao
